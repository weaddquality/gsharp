G# Templates

There are a number of different templates that can be used when creating classes.

·         G#Template – Standard template for creating classes. Base or sub class. Recommended.

·         G#Interface – Standard template for creating an interface.

·         G#StaticTemplate – Template when creating a static class with only static member. Can not be inherited.

·         LvNative – LabVIEW native class. Not reference based.

·         G#InterfaceImplementation – Template used when implementing an interface.

·         G#BaseTemplateG3 – Template used for creating a base class compatible with Endevo GOOP3. Used when converting classes from GOOP3 to G#. Not recommended when creating new classes.

·         G#SubTemplateG3 – Template used for creating a sub class compatible with Endevo GOOP3. Used when converting classes from GOOP3 to G#. Not recommended when creating new classes. 

 

All templates are located in:

<LabVIEW>/vi.lib/addons/_AddQ/G#Templates

G# Classes File Structure

All G# classes have the same structure as shown below. A G# class is located in a folder called <ClassName>_class. With the following structure.

·         <top> - lvclass-file and public methods

·         protected – protected methods and attribute definitions type def controls

·         private – private methods of the class

·         properties – properties methods of the class. If using LabVIEW 2010 these will contain the new property definition folder.

·         utils – G# implementation support VIs and controls. The GetAttributes and GetClassAttributes VIs for accessing the attributes are located here.

 G# Object Attributes Data

The object attributes are updated with in place element structure. The G# Object use ByRef with Data Value Reference (DVR) to keep attribute data. Use in place element structure like picture and connect DVR from method .._GetAttributes.vi in utils to access object data.

 Property Definition

The property definition folder will be used to contain the properties. In the code, you can use LabVIEW property nodes on the class reference to access the attributes through property definition methods.

G# Wire Appearance

The G# uses two kinds of wire appearance.

 

G#Object – all G# object references get this wire appearance, a solid green line.

G#Interface –all G# Interface references get this wire appearance.
